# CoTabor 飞书后端说明（兼容/历史路径）

[English](../../en/manual/feishu.md) | 简体中文

仓库里仍保留飞书相关后端代码，但它已经不是当前顶层 Options UI 的主文档路径。这个文档应视为兼容说明，主要给仍在维护旧飞书链路的维护者使用。

## 当前状态

- 当前顶层 Options UI 主要暴露的是 `Notion`、`LLM`、`MCP`
- 飞书已经不是 README 推荐的默认后端路径
- 如果继续维护飞书链路，应先以当前代码实现为准，再决定是否沿用旧 UI 操作步骤

## 第一步：飞书扫码授权

1. 先确认你当前维护的分支是否仍然暴露飞书设置入口，或存在等价的维护者专用接入流程。
2. 打开 CoTabor 插件的选项设置页。
3. 使用当前构建里可用的飞书授权入口。
4. 在飞书网页授权流程中完成扫码确认。
5. 在界面或日志中确认授权状态已经被正确保存。

> *提示：CoTabor 会获取并使用合规的飞书官方授权 Token 进行接口调用，安全可靠，不接触任何其它个人信息。*

## 第二步：构建 AI 记忆仓库

由于底层调用方式，需要创建一个用来存放配置与数据的空文件夹：

1. 打开飞书云文档空间：[飞书云文档](https://feishu.cn/drive/)。
2. 在适合的地方（比如“我的空间”根目录）创建一个 **新的空文件夹**，并命名为类似 "CoTabor_Memories"。
3. 复制该文件夹的完整网页链接（类似 `https://xxx.feishu.cn/drive/folder/xxx...`）。
4. 切回 CoTabor 设置页，在“步骤 2：构建多维表格”将刚才得到的 **文件夹链接（或 Folder Token）粘贴到输入框中**。
5. 点击 **一键初始化 AI 数据中心**。

> 稍等几秒钟，系统应当会在你指定的文件夹下创建用于执行日志与记忆沉淀的多维表格数据库底座。最终结果请以当前分支的实际行为为准。

## 维护说明

如果当前构建已经没有对应的飞书 UI 接入入口，就不要继续在这里补“推测出来”的操作步骤。应先恢复或重构产品入口，再更新这份文档。
